# How I checked the calculations

I did not want the Black–Scholes implementation to be checked only against values generated by the same code. The checks therefore mix known benchmarks, independent finite differences, accounting identities, and reproducible simulations.

## Automated checks

The suite contains 41 tests in four groups:

- payoffs, bounds, and parity;
- Black–Scholes pricing and Greeks;
- implied-volatility inversion and surface diagnostics;
- dynamic hedging and P&L attribution.

## Independent references and identities

| Component | Check used |
|---|---|
| Pricing | Known benchmark values, no-arbitrage bounds, put–call parity |
| Greeks | Central finite differences and call/put identities |
| Implied volatility | Recovery of known generating volatility with three solvers |
| Surface | Recovery of generating IV; strike and calendar consistency checks |
| Path simulation | Positivity, array shape, and fixed-seed reproducibility |
| Hedge ledger | Closed-form one-period example and exact cash reconciliation |
| Position signs | Long/short symmetry without transaction costs |
| Costs | Terminal drag equals the future value of paid transaction costs |
| Attribution | Exact counterfactual reconstruction and Greek-ledger reconciliation |

## Reproduction commands

```bash
python -m pip install -e ".[dev]"
pytest
python scripts/run_experiments.py
```

## Edge cases I treated explicitly

- zero time to expiration;
- zero volatility;
- prices at or outside no-arbitrage bounds;
- low-vega implied-volatility inversion;
- scalar and vectorized pricing inputs;
- final stock liquidation and option settlement;
- initial, intermediate, and terminal transaction costs.

## Interpretation boundary

The saved Monte Carlo runs are intentionally limited in size. They are sufficient to check the mechanics and accounting identities shown in the repository, but they are not estimates of a live strategy's performance.
